## Critical build/workflow facts
- **Production build MUST be `next build --webpack`** — Next 16's default Turbopack build fails collecting route-group pages. Pinned in `vercel.json` (`buildCommand`). Don't "clean up" the `--webpack` flag.
- **Migrations are run by the human** by pasting the `.sql` into the Supabase SQL Editor. NEVER via CLI or Claude-in-Chrome (auto-pairing corrupts `$`/`{}`/`[]`). I write migration files; I do not apply them.
- **Always push code changes to `main`** automatically once verified (Vercel deploys on push). Do NOT push/apply DB migrations.
- Risky/auth changes → **branch + Vercel preview + test before merging.**
- Gates: `npx tsc --noEmit`, `npm run lint` (ESLint 9 flat config; ~8 advisory React-Compiler warnings, **0 errors** is the bar), `npm test` (21 vitest), `npm run build`.
- `npm audit`: down to **2 moderate** (PostCSS via Next's bundled copy; only fixed by a future Next patch / Next 15+). Not a blocker.

## Permission-based Calendar — SHIPPED to `main` (migration 038 PENDING)
Shared team calendar at `/calendar` (a SHARED_ROUTES route — all roles allowed by the layout guard, then gated in-page). Access: surveyors + admins always; office only with the new `calendar.view` permission; clients blocked.
- **Two sources:** (1) **jobs** read live via SECURITY DEFINER `get_calendar_jobs(start,end)` — returns only safe scheduling fields (title, job_number, status, scheduled_date, vessel, surveyor, client) for the WHOLE team (product decision: everyone sees the whole job schedule; internal_notes etc. NOT exposed). (2) **`calendar_events`** table for leave + general events.
- **Visibility (RLS, migration 038):** *leave* = owner + admins ONLY (other surveyors see nothing); surveyors file their own as `pending`, admins approve/reject (reuses the review pattern). *general* events = admin-created with audience `everyone` / `roles` (visible_roles) / `users` (visible_user_ids). RLS SELECT enforces all of it; owners may edit/cancel their own pending leave; admins manage everything.
- **UI:** month grid (date-fns, hand-rolled, no new deps), day-detail modal, "Request leave" (surveyor/admin), admin "Add event" with colour + audience picker (role chips / searchable people picker), and a pending-leave approvals panel for admins. Live via `useRealtimeRefresh('calendar_events')`. Nav: Calendar added to admin + surveyor sidebars and office (when `calendar.view` granted).
